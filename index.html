What is Steganography?

Steganography is the art of hiding secret information inside ordinary files.

Unlike encryption which scrambles data, steganography conceals the very existence of a message — hiding it in plain sight within the pixel structure of an image.

Frequently Asked Questions

General

What is steganography?
Steganography is the art of hiding secret messages inside ordinary files like images, audio, or video. Unlike encryption which scrambles data, steganography conceals the existence of the message itself, making it invisible to casual observers.
How does this tool work?
Our tool uses the LSB (Least Significant Bit) technique. It modifies the least significant bits of pixel RGB values to store your message. These tiny changes are invisible to the human eye but can be extracted later to reveal the hidden text.
Is my data safe?
Yes. All processing happens entirely in your browser using JavaScript. No images or messages are ever sent to any server. Your data never leaves your device, ensuring complete privacy.

Technical

What image formats are supported?
Only PNG format is supported. PNG is a lossless format, meaning pixel data remains unchanged after saving. This is crucial for steganography — any lossy compression would corrupt the hidden message.
What is the maximum message length?
The current limit is 500 characters. This also depends on image dimensions — a larger image can accommodate more data. A standard 1920×1080 PNG can comfortably hold 500–800 characters.
Can I use JPEG images?
No. JPEG uses lossy compression which alters pixel values during the save process, destroying any hidden messages. You must use PNG format for both encoding and decoding.
How do I decode a message?
Go to the Decode page, upload the encoded PNG file you received, and click Extract Message. The hidden text will appear in the output panel and can be copied to your clipboard.

Troubleshooting

Why can't I extract the message?
Ensure you are using the exact PNG file downloaded after encoding. If the image was converted to JPEG, re-saved, edited, or compressed at any point, the hidden data will be lost. Always preserve the original downloaded file.
The image is too large. What should I do?
Use a smaller or resized image. For most use cases, images around 1920×1080 work well. Always ensure the resized file is saved in PNG format before encoding.
My browser shows an error.
Ensure you are using a modern browser — Chrome, Firefox, Edge, or Safari are all supported. Clear your browser cache and retry. If the issue persists, try a different browser.
